Build a shared UI component library and admin_page_config system to standardize every admin page. This is the foundation sprint — all subsequent sprints depend on these components.

(1) Create an admin_page_config table in the Drizzle schema (shared/schema.ts):
- id: uuid, primary key, default gen_random_uuid()
- page_key: text, unique, not null (e.g., 'dashboard', 'content-factory', 'monetization', 'network', 'community', 'analytics')
- title: text, not null
- description: text (used as tooltip on hover)
- icon_name: text, not null (Lucide icon name like 'LayoutDashboard', 'FileText', 'DollarSign', 'Users2', 'Users', 'BarChart3')
- primary_action_label: text, nullable (e.g., '+ New Content', '+ New Campaign')
- ai_action_label: text, nullable (e.g., 'AI Generate', 'AI Optimize', 'AI Analyze')
- metrics_config: jsonb, nullable
- sort_order: integer, not null
- is_visible: boolean, default true
